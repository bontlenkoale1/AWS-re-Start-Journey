# Using Amazon Inspector for vulnerability assesment and remediation
---

## 📌 Overview

In this lab, I used **Amazon Inspector** to scan AWS resources for vulnerabilities, specifically **AWS Lambda functions**.

The goal was to understand how automated security tools can detect vulnerable software packages and help remediate them quickly.

In this scenario, **AnyCompany** is developing an application using **AWS Lambda**. During development, the team needs a security solution that automatically scans deployed resources for vulnerabilities.

To address this need, I activated **Amazon Inspector**, which continuously scans:

* **AWS Lambda functions**
* **Amazon EC2 instances**
* **Amazon ECR container images**

Amazon Inspector automatically identifies vulnerabilities in packages and code and provides remediation recommendations.

---

# 🎯 Objectives

By completing this lab, I was able to:

* Activate **Amazon Inspector**
* Analyze and interpret vulnerability findings
* Remediate vulnerabilities detected in **AWS Lambda functions**

# 🧪 Lab Environment

The lab environment contained:

* Pre-configured **AWS Lambda functions**
* Vulnerable software packages intentionally included for testing
* **Amazon Inspector** used to scan and report vulnerabilities by severity

# Task 1️⃣: Activating Amazon Inspector

First, I activated **Amazon Inspector** so it could start scanning resources in the AWS account.

### Steps I followed

1. Opened the **AWS Management Console**
2. Searched for **Amazon Inspector**
3. Selected **Activate Inspector**
4. Clicked **Activate Inspector** to enable scanning

Once activated, Amazon Inspector automatically began scanning resources in the environment.

### Resources scanned by default

* Amazon EC2
* Amazon ECR
* AWS Lambda

After activation, the dashboard showed that **Lambda function scanning was enabled**.

---

# Task 2️⃣: Reviewing Vulnerability Findings

After activating Amazon Inspector, I reviewed the **security findings** generated by the scan.

### Steps I followed

1. Navigated to **Findings → All findings**
2. Reviewed the vulnerabilities detected in Lambda functions
3. Selected the finding:

**CVE-2023-32681 – requests**

### Key information shown in the report

* **Severity:** Medium
* **Impacted Resource:** Lambda function
* **Vulnerability:** Outdated `requests` Python package

Amazon Inspector also provided:

* Detailed vulnerability information
* A link to the **National Vulnerability Database (NVD)**
* Recommended remediation steps

# Task 3️⃣: Remediating the Vulnerability

The vulnerability existed because the Lambda function was using an **outdated version of the `requests` Python package**.

### Steps I followed

1. Opened **AWS Lambda**
2. Selected the function:

```
get-request
```

3. Opened the file:

```
requirements.txt
```

The file originally contained:

```
requests==2.20.0
```

4. I removed the version number so it became:

```
requests
```

By removing the version constraint, Lambda installs the **latest secure version** of the package.

5. Clicked **Deploy** to update the Lambda function.

After deployment, **Amazon Inspector automatically triggered a new scan**.

# ✅ Verifying the Remediation

After the new scan completed:

1. I returned to **Amazon Inspector**
2. Navigated to **Findings**
3. Changed the filter from **Active → Closed**

The vulnerability:

```
CVE-2023-32681 – requests
```

was now listed as **Closed**, confirming that the issue had been successfully remediated.
